Explications des commandes Git utilisées

=== ÉTAPE 1 - CRÉATION DU PROJET ===

git add histoire.txt
Ajoute le fichier à la zone de staging pour le préparer au commit.

git commit -m "Ajout du chapitre 1"
Enregistre une version du projet avec le message donné.

=== ÉTAPE 2 - AJOUT DE CONTENU ===

echo "Chapitre 2..." >> histoire.txt
Ajoute du texte à la fin du fichier existant.

git commit (pour chaque chapitre)
Crée un nouveau commit pour sauvegarder chaque ajout.

=== ÉTAPE 3 - EXPLORATION DE L'HISTORIQUE ===

git log
Affiche l'historique complet : hash, auteur, date et message de chaque commit.

git log --oneline
Version condensée de l'historique, une ligne par commit, plus facile à lire.

git show <hash>
Montre les détails d'un commit précis avec les lignes modifiées.

git diff HEAD~2 HEAD
Compare deux versions du fichier en montrant ce qui a changé entre elles.

=== ÉTAPE 4 - CORRIGER LE DERNIER COMMIT ===

git commit --amend -m "nouveau message"
Modifie le message du dernier commit sans créer un nouveau commit.

=== ÉTAPE 5 - REVENIR EN ARRIÈRE ===

git reset --soft HEAD~1
Annule le dernier commit mais garde les modifications dans la zone de staging.

cat histoire.txt
Affiche le contenu du fichier pour vérifier qu'il n'a pas été supprimé.

=== ÉTAPE 6 - ANNULER UN ANCIEN COMMIT ===

git revert <hash>
Crée un nouveau commit qui annule les changements d'un ancien commit sans supprimer l'historique.

Note : Dans mon cas, ça a créé un conflit. J'ai dû éditer manuellement histoire.txt 
pour supprimer le chapitre 2, puis finir avec git add et git commit.

=== ÉTAPE 7 - SAUVEGARDER TEMPORAIREMENT ===

git stash
Met de côté les modifications en cours sans les commiter. Le fichier revient à l'état du dernier commit.

git stash pop
Récupère les modifications mises de côté et les réapplique dans le fichier.

=== VÉRIFICATIONS FINALES ===

git status
Vérifie l'état du dépôt (fichiers modifiés, stagés, etc.).

git log --oneline
Consulte l'historique final pour vérifier tous les commits.

=== COMMANDES BONUS (PUSH) ===

git remote add origin <url>
Lie le dépôt local à un dépôt distant sur GitHub.

git push -u origin main
Envoie tous les commits locaux vers le dépôt GitHub.
